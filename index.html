Introducing Today's Project!

In this project, I'm going to set up Claude Code and use it to fix missing documentation across the Automattic Hub — an internal WordPress enablement platform serving 6 business units. This will help me understand how AI coding agents work in a real production codebase. I'm interested in this because Claude Code is how the best engineers are working in 2026.

Key tools and concepts

The key tools I used include Claude Code CLI, CLAUDE.md, git, and the Automattic Hub WordPress codebase. Key concepts I learnt include agentic AI, project context windows, and trust permissions.

Installing Claude Code

In this step, I'm going to install Claude Code globally using npm and authenticate with my Anthropic API key. Claude Code is an agentic AI coding tool — it lives in your terminal and can read files, run commands, and push code autonomously.

Claude Code vs Claude.ai

I installed Claude Code by running npm install -g @anthropic-ai/claude-code in Terminal. Claude Code is different from Claude.ai because it's an agent — it has direct access to your filesystem, can run bash commands, and make real changes without you clicking anything.

Understanding the Trust Prompt

When I launched Claude Code outside a project folder, it asked whether I trusted the current directory. I chose not to accept immediately — I wanted to understand exactly what file permissions I was granting before giving an AI agent access to my machine.

Why a project folder matters
Without a project folder, Claude Code gives generic responses. Inside a real project it can see your full architecture, read your actual files, and give precise help tailored to your specific codebase.

Setting Up Project Memory with CLAUDE.md

CLAUDE.md is a special file Claude Code reads at the start of every session — it's the persistent memory layer for your project. I edited it to include the platform name, stack, and the specific task I wanted Claude Code to solve.

CLAUDE.md
1# Project Context
2 
3## What this project is
4  My Learning Project — a hands-on AI skills portfolio
5  built with Claude Code and NextWork.
6 
7## Current task
8  Build and document an AI project from scratch,
9  generate portfolio documentation automatically.

How CLAUDE.md works

CLAUDE.md is a markdown file at the root of your project. This is useful because every session starts with full context — no re-explaining your project architecture each time you open Claude Code.

Building and Iterating on the Portfolio

I asked Claude Code to audit and fix the missing documentation templates across three product pages. It read the existing pages, identified the gaps, wrote the missing content, staged the changes, and pushed to origin/dev in a single session.
